把记录留在自己手里

InkRoot 的核心不是把笔记变复杂，而是让记录可以稳定留下来，并且能按你的方式同步、备份和迁移。

本地优先

没有网络也能写。笔记、图片和设置优先保存在设备上，适合灵感、摘录、待办和日常备忘。

连接自部署 Memos

支持官方服务器，也支持自部署 Memos。兼容 Memos 0.21.0 之后的常见登录、同步、资源和引用接口差异。

备份自己掌握

WebDAV 可备份笔记和图片附件，适合把资料放到自己的云盘或 NAS。导入导出也保留迁移路径。

从任何入口写下来

很多笔记不是在打开 App 后才出现的。InkRoot 围绕手机和桌面的真实入口设计，让记录尽量少被流程打断。

系统分享

从浏览器、阅读器、相册、文件等入口接收文本、链接、图片和附件，减少复制粘贴。

小组件

快速记录和随机回顾分成两个组件。随机回顾展示真实笔记快照，适合把旧记录重新带回视野。

离线发布

网络不稳定时先保存，恢复连接后再同步，适合自部署服务不在同一网络的场景。

图片与附件

支持图片上传、预览、保存和多图浏览，图片附件也可以随 WebDAV 备份。

整理、搜索和回顾一起工作

好的笔记工具不只负责保存，还要帮你找回上下文。InkRoot 把标签、搜索、相关笔记、随机回顾和知识图谱放在同一套记录体验里。

多级标签

支持类似 #工作/项目A 的层级标签，适合把长期主题、项目和资料来源分层管理。

搜索和相关笔记

通过全文搜索找到明确内容，通过相关笔记连接相近上下文，避免孤立地看单条记录。

随机回顾和知识图谱

随机回顾让旧笔记重新出现，知识图谱用于观察标签、内容和笔记之间的关系。

迁移、扩展和 AI 都不强绑

你可以从已有工具迁移，也可以只使用本地记录。AI 是可选能力，模型和接口由用户自行配置。

导入迁移

支持 Flomo、微信读书和本地备份等导入路径，降低从旧工具迁移的成本。

可选 AI

摘要、续写、标签推荐、点评和洞察使用用户自填 API 地址、模型名和 Key，不把模型选择写死。

跨平台构建

发布流程覆盖 Android、iOS unsigned、macOS、Windows 和 Linux，桌面端和移动端尽量保持一致体验。

当前支持的平台

发布包通过 GitHub Release 提供。国内用户可在下载页使用加速链接，完整历史版本保留在更新日志中。
