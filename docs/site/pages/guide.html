Guide

使用指南

先选使用模式，再配置同步和备份。InkRoot 可以只做本地笔记，也可以作为 Memos 客户端使用。

1. 选择使用模式

本地模式

适合先记录、后整理。无需登录，笔记保存在设备本地。建议定期导出或开启 WebDAV 备份。

官方服务器

适合想快速同步的用户。登录官方服务器后，笔记和附件会同步到官方服务。

自部署 Memos

适合已有 Memos 服务的用户。服务器地址可以是域名，也可以是内网 IP + 端口；如果使用 HTTP，请确认设备能访问该地址。

2. 登录 Memos 或官方服务器

  1. 进入设置中的服务器或账号入口。
  2. 填写服务器地址、账号和密码。
  3. 如果是自部署服务，确认浏览器也能打开同一地址。
  4. 登录后下拉刷新或点击同步，确认笔记能正常读取。

InkRoot 会自动尝试兼容 Memos 0.21.0 之后的常见 API 差异。如果某个自部署版本仍无法登录，请在 GitHub Issue 中提供版本号、服务器地址格式和错误提示。

3. 配置 WebDAV 备份

WebDAV 用于把本地数据备份到你自己的云盘或 NAS。当前支持笔记数据和图片附件备份，可在设置中选择是否备份图片。

  • 先测试连接，再执行备份。
  • 坚果云等服务通常需要应用专用密码。
  • 恢复前建议先导出当前数据，避免误覆盖。

4. 导入与迁移

InkRoot 提供 Flomo、微信读书和本地备份等导入路径。导入前建议保留原始导出文件，导入后再检查标签、图片和换行排版是否符合预期。

  • Flomo 导入适合迁移历史闪念笔记。
  • 微信读书导入适合整理书摘和阅读想法。
  • 本地备份适合设备迁移或恢复前留存快照。

5. 使用系统分享

Android 支持从浏览器、阅读器、相册、文件管理器等入口分享文本、网页、图片和文件到 InkRoot。iOS 分享扩展已进入构建链路，真机表现以系统分享面板为准。

6. 使用桌面小组件

小组件分为记录和随机回顾两类。随机回顾显示的是应用生成的笔记快照；如果刚安装或刚恢复数据，先打开 InkRoot 同步一次小组件。

7. 配置 AI

AI 功能不是必需项。需要时在设置中填写 API 地址、模型名和 API Key。模型名使用输入框，是为了适配快速变化的模型列表。

  • 摘要：为长笔记生成简短概括。
  • 续写：基于当前内容继续补充。
  • 标签推荐：从内容中建议标签。
  • 点评和洞察：尽量给出和当前笔记相关的具体反馈。
