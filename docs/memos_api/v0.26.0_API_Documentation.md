## 📋 版本概述

v0.26.0 是 Memos 的重大架构升级版本，实现了**认证机制重构**、**React Query 迁移**和**角色系统重构**。该版本专注于提升安全性、性能和开发者体验。

### 🎯 核心变更
- 🔐 **刷新令牌轮换**：滑动窗口会话机制
- ⚡ **React Query v5 迁移**：前端状态管理现代化
- 👥 **角色重构**：HOST → ADMIN
- 🎨 **HDR 媒体支持**：HDR 图像和视频
- 🗺️ **地理位置可视化**：Google Maps 集成
- 📡 **Connect-RPC**：从 REST 迁移到 gRPC

### ⚠️ Breaking Changes
- **移除遗留 Session Cookie 认证**
- **角色名称变更**：HOST → ADMIN（自动迁移）
- **视频/音频流式传输**：修复内存耗尽问题
- **EXIF 元数据剥离**：上传图片自动去除位置信息

## 🔐 认证机制（重构）

### 刷新令牌轮换（Refresh Token Rotation）

v0.26.0 实现了业界标准的刷新令牌轮换机制，提供更高的安全性。

**认证流程**

```
用户登录
  ↓
返回 Access Token + Refresh Token
  ↓
Access Token 短期有效（15分钟）
Refresh Token 长期有效（7天）
  ↓
Access Token 过期时用 Refresh Token 刷新
  ↓
返回新的 Access Token + 新的 Refresh Token（旧的失效）
```

### 登录 API

**请求**
```http
POST /api/v1/auth/signin
Content-Type: application/json

{
  "username": "your_username",
  "password": "your_password"
}
```

**响应**
```json
{
  "user": {
    "id": 1,
    "username": "your_username",
    "role": "ADMIN",
    "email": "user@example.com",
    "nickname": "Your Name"
  },
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresAt": 1234567890
}
```

### 刷新 Token

**请求**
```http
POST /api/v1/auth/refresh
Content-Type: application/json

{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**响应**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresAt": 1234567890
}
```

**重要特性**
- ✅ 旧 Refresh Token 使用一次后立即失效
- ✅ 跨标签页令牌同步（localStorage）
- ✅ Refresh Cookie 回退机制
- ✅ 滑动窗口会话管理

### 使用 Access Token

所有 API 请求需携带 Access Token：

```http
GET /api/v1/memo
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 登出 API

**请求**
```http
POST /api/v1/auth/signout
Authorization: Bearer <access_token>
```

**响应**
```json
{
  "success": true
}
```

**行为**
- 撤销当前 Access Token
- 撤销关联的 Refresh Token
- 清除服务器端会话

新版本使用 gRPC 风格的错误响应：

**认证错误**
```json
{
  "code": "UNAUTHENTICATED",
  "message": "Token已过期或无效"
}
```

**权限错误**
```json
{
  "code": "PERMISSION_DENIED",
  "message": "无权限访问此资源"
}
```

**资源不存在**
```json
{
  "code": "NOT_FOUND",
  "message": "备忘录不存在"
}
```

**参数错误**
```json
{
  "code": "INVALID_ARGUMENT",
  "message": "content字段不能为空"
}
```

### gRPC 状态码映射

| HTTP 状态码 | gRPC Code | 说明 |
|------------|-----------|------|
| 200 | OK | 成功 |
| 400 | INVALID_ARGUMENT | 参数错误 |
| 401 | UNAUTHENTICATED | 未认证 |
| 403 | PERMISSION_DENIED | 无权限 |
| 404 | NOT_FOUND | 资源不存在 |
| 409 | ALREADY_EXISTS | 资源已存在 |
| 500 | INTERNAL | 服务器内部错误 |

## 💡 最佳实践

### 1. Token 刷新策略

```javascript
class AuthService {
  constructor() {
    this.accessToken = null;
    this.refreshToken = null;
    this.tokenExpiresAt = null;
  }

  async login(username, password) {
    const response = await fetch('/api/v1/auth/signin', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ username, password })
    });

    const data = await response.json();
    this.saveTokens(data);
    return data.user;
  }

  saveTokens({ accessToken, refreshToken, expiresAt }) {
    this.accessToken = accessToken;
    this.refreshToken = refreshToken;
    this.tokenExpiresAt = expiresAt;

    // 持久化到 localStorage
    localStorage.setItem('accessToken', accessToken);
    localStorage.setItem('refreshToken', refreshToken);
    localStorage.setItem('tokenExpiresAt', expiresAt);
  }

  async refreshAccessToken() {
    try {
      const response = await fetch('/api/v1/auth/refresh', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ refreshToken: this.refreshToken })
      });

      if (!response.ok) {
        throw new Error('Refresh token expired');
      }

      const data = await response.json();
      this.saveTokens(data);
      return data.accessToken;
    } catch (error) {
      // Refresh token 也过期了，需要重新登录
      this.logout();
      throw error;
    }
  }

  async request(url, options = {}) {
    // 检查 token 是否即将过期（提前 5 分钟刷新）
    const now = Date.now();
    const expiresIn = this.tokenExpiresAt - now;

    if (expiresIn < 5 * 60 * 1000) {
      await this.refreshAccessToken();
    }

    // 携带 Access Token 发起请求
    const response = await fetch(url, {
      ...options,
      headers: {
        ...options.headers,
        'Authorization': `Bearer ${this.accessToken}`
      }
    });

    // 处理 401 错误
    if (response.status === 401) {
      // 尝试刷新 token
      await this.refreshAccessToken();
      // 重试原请求
      return fetch(url, {
        ...options,
        headers: {
          ...options.headers,
          'Authorization': `Bearer ${this.accessToken}`
        }
      });
    }

    return response;
  }

  logout() {
    this.accessToken = null;
    this.refreshToken = null;
    this.tokenExpiresAt = null;
    localStorage.clear();
  }
}
```

## 🐛 已知问题与解决方案

### Token 刷新竞态条件

**问题**：多个请求同时刷新 token 可能导致冲突。

**解决**：使用单例模式和锁机制
```javascript
class TokenRefreshLock {
  constructor() {
    this.refreshPromise = null;
  }

  async refresh(refreshFn) {
    if (!this.refreshPromise) {
      this.refreshPromise = refreshFn().finally(() => {
        this.refreshPromise = null;
      });
    }
    return this.refreshPromise;
  }
}

const lock = new TokenRefreshLock();
await lock.refresh(() => authService.refreshAccessToken());
```

### 从 v0.25.3 升级权限问题

**问题**：升级后出现权限错误。

**解决**：v0.26.0 包含自动修复脚本，首次启动时自动运行。
