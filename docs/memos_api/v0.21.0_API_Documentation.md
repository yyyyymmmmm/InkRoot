## 📋 版本概述

v0.21.0 是 Memos 的基础稳定版本，提供了完整的备忘录管理核心功能。该版本使用传统的 REST API 架构，采用 Session Cookie 认证方式。

### 核心特性
- ✅ 完整的备忘录 CRUD 操作
- ✅ 用户认证和权限管理
- ✅ 标签系统
- ✅ 附件管理
- ✅ Markdown 渲染
- ✅ 基础搜索和过滤
- ✅ RSS 订阅

## 🔐 认证机制

### Session Cookie 认证

**登录端点**
```http
POST /api/auth/signin
Content-Type: application/json

{
  "username": "your_username",
  "password": "your_password"
}
```

**响应**
```json
{
  "id": 1,
  "username": "your_username",
  "role": "HOST",
  "email": "user@example.com",
  "nickname": "Your Name",
  "createdTs": 1234567890,
  "updatedTs": 1234567890
}
```

认证成功后，服务器会设置 Session Cookie，后续请求自动携带。

### Access Token (JWT)

**创建 Token**
```http
POST /api/user/token
Cookie: session_token=xxx

{
  "description": "My API Token"
}
```

**响应**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "description": "My API Token",
  "createdTs": 1234567890
}
```

**使用 Token**
```http
GET /api/memo
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 📝 备忘录 API

### 创建备忘录

**请求**
```http
POST /api/memo
Content-Type: application/json
Cookie: session_token=xxx

{
  "content": "This is a memo #tag",
  "visibility": "PRIVATE"
}
```

**参数说明**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `content` | string | ✅ | 备忘录内容（Markdown 格式） |
| `visibility` | string | ❌ | 可见性：`PRIVATE`, `PROTECTED`, `PUBLIC` |
| `resourceIdList` | array | ❌ | 附件 ID 列表 |

**响应**
```json
{
  "id": 100,
  "creatorId": 1,
  "createdTs": 1234567890,
  "updatedTs": 1234567890,
  "content": "This is a memo #tag",
  "visibility": "PRIVATE",
  "pinned": false,
  "resourceList": [],
  "relationList": []
}
```

### 查询备忘录列表

**请求**
```http
GET /api/memo?creatorId=1&rowStatus=NORMAL&limit=10&offset=0
Cookie: session_token=xxx
```

**查询参数**
| 参数 | 类型 | 说明 |
|------|------|------|
| `creatorId` | integer | 创建者 ID |
| `rowStatus` | string | 状态：`NORMAL`, `ARCHIVED` |
| `limit` | integer | 每页数量（默认 10） |
| `offset` | integer | 偏移量（默认 0） |
| `tag` | string | 按标签过滤 |
| `visibility` | string | 按可见性过滤 |

**响应**
```json
[
  {
    "id": 100,
    "creatorId": 1,
    "createdTs": 1234567890,
    "updatedTs": 1234567890,
    "content": "This is a memo #tag",
    "visibility": "PRIVATE",
    "pinned": false,
    "resourceList": [],
    "relationList": []
  }
]
```

### 获取单个备忘录

**请求**
```http
GET /api/memo/100
Cookie: session_token=xxx
```

**响应**
```json
{
  "id": 100,
  "creatorId": 1,
  "createdTs": 1234567890,
  "updatedTs": 1234567890,
  "content": "This is a memo #tag",
  "visibility": "PRIVATE",
  "pinned": false,
  "resourceList": [
    {
      "id": 1,
      "filename": "image.png",
      "externalLink": "",
      "type": "image/png",
      "size": 12345,
      "createdTs": 1234567890
    }
  ],
  "relationList": []
}
```

### 更新备忘录

**请求**
```http
PATCH /api/memo/100
Content-Type: application/json
Cookie: session_token=xxx

{
  "content": "Updated content",
  "visibility": "PUBLIC",
  "pinned": true
}
```

**可更新字段**
- `content`: 备忘录内容
- `visibility`: 可见性
- `pinned`: 是否置顶
- `resourceIdList`: 附件列表
- `rowStatus`: 状态（归档/正常）

**响应**
```json
{
  "id": 100,
  "content": "Updated content",
  "visibility": "PUBLIC",
  "pinned": true,
  "updatedTs": 1234567891
}
```

### 删除备忘录

**请求**
```http
DELETE /api/memo/100
Cookie: session_token=xxx
```

**响应**
```json
{
  "success": true
}
```

### 归档/恢复备忘录

**归档**
```http
PATCH /api/memo/100
Content-Type: application/json

{
  "rowStatus": "ARCHIVED"
}
```

**恢复**
```http
PATCH /api/memo/100
Content-Type: application/json

{
  "rowStatus": "NORMAL"
}
```

## 💡 最佳实践

### 1. 认证方式选择

**Web 应用**：使用 Session Cookie
```javascript
// 登录
fetch('/api/auth/signin', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include',
  body: JSON.stringify({ username, password })
});

// 后续请求自动携带 Cookie
fetch('/api/memo', { credentials: 'include' });
```

**移动应用/第三方集成**：使用 Access Token
```javascript
// 获取 Token
const { accessToken } = await fetch('/api/user/token', {
  method: 'POST',
  credentials: 'include',
  body: JSON.stringify({ description: 'Mobile App' })
}).then(r => r.json());

// 使用 Token
fetch('/api/memo', {
  headers: { 'Authorization': `Bearer ${accessToken}` }
});
```

### 2. 分页查询

```javascript
// 首次加载
let memos = await fetch('/api/memo?limit=20&offset=0').then(r => r.json());

// 加载更多
let moreMemos = await fetch('/api/memo?limit=20&offset=20').then(r => r.json());
```

### 3. 附件处理

```javascript
// 上传附件
const formData = new FormData();
formData.append('file', fileInput.files[0]);

const resource = await fetch('/api/resource', {
  method: 'POST',
  credentials: 'include',
  body: formData
}).then(r => r.json());

// 创建带附件的备忘录
await fetch('/api/memo', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    content: 'Check this image: ![image](/o/r/' + resource.id + '/' + resource.filename + ')',
    resourceIdList: [resource.id]
  })
});
```

### 4. 标签提取

```javascript
// 从内容中提取标签
function extractTags(content) {
  const tagRegex = /#([^\s#]+)/g;
  const tags = [];
  let match;
  while ((match = tagRegex.exec(content)) !== null) {
    tags.push(match[1]);
  }
  return [...new Set(tags)]; // 去重
}

const content = "This is a memo #work #important";
const tags = extractTags(content); // ["work", "important"]
```

### 5. 错误处理

```javascript
async function createMemo(content) {
  try {
    const response = await fetch('/api/memo', {
      method: 'POST',
      credentials: 'include',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ content })
    });

    if (!response.ok) {
      if (response.status === 401) {
        throw new Error('Not authenticated');
      } else if (response.status === 403) {
        throw new Error('Permission denied');
      }
      throw new Error('Failed to create memo');
    }

    return await response.json();
  } catch (error) {
    console.error('Create memo failed:', error);
    throw error;
  }
}
```

## 🚀 快速开始示例

### 完整工作流示例

```javascript
// 1. 登录
await fetch('/api/auth/signin', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include',
  body: JSON.stringify({
    username: 'demo',
    password: 'demo'
  })
});

// 2. 创建备忘录
const memo = await fetch('/api/memo', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    content: 'My first memo #demo',
    visibility: 'PRIVATE'
  })
}).then(r => r.json());

console.log('Created memo:', memo.id);

// 3. 上传图片
const formData = new FormData();
formData.append('file', imageFile);

const resource = await fetch('/api/resource', {
  method: 'POST',
  credentials: 'include',
  body: formData
}).then(r => r.json());

// 4. 更新备忘录，添加图片
await fetch(`/api/memo/${memo.id}`, {
  method: 'PATCH',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    content: `My first memo with image #demo\n\n![image](/o/r/${resource.id}/${resource.filename})`,
    resourceIdList: [resource.id]
  })
});

// 5. 查询所有备忘录
const memos = await fetch('/api/memo?limit=10&offset=0', {
  credentials: 'include'
}).then(r => r.json());

console.log('Total memos:', memos.length);

// 6. 按标签过滤
const demoMemos = await fetch('/api/memo?tag=demo', {
  credentials: 'include'
}).then(r => r.json());

console.log('Demo memos:', demoMemos.length);
```

## 🔄 迁移到新版本

v0.21.0 是稳定的基础版本，升级到更高版本需要注意：

### 升级到 v0.24.0+
- 新增快捷方式功能
- 需要数据库迁移
- API 保持向后兼容

### 升级到 v0.26.0+
- 认证机制重构（刷新令牌轮换）
- HOST 角色重命名为 ADMIN
- 需要重新配置认证流程

### 升级到 v0.27.0+
- 用户资源名称从 ID 改为用户名
- 部分 API 端点路径变更
- 新增 SSE 实时更新功能
